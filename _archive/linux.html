Linux File System Overview

/                    (Root - Top of hierarchy)
├── home/            (User home directories)
│   └── biouser/     (Your personal space)
├── bin/             (Essential binaries/commands)
├── etc/             (Configuration files)
├── var/             (Variable data - logs, databases)
│   └── log/         (System and application logs)
├── tmp/             (Temporary files)
├── usr/             (User programs and data)
│   ├── bin/         (User binaries)
│   └── local/       (Locally installed software)
└── opt/             (Optional software packages)
Biotech Context: Bioinformatics tools often install in /usr/local or /opt. Your sequence data might be in /home/biouser/data or /var/data
📝 Speaker Notes:
The Linux file system follows a hierarchical tree structure starting from root (/). Unlike Windows (C:, D:), everything is under one root. For biotech work: /home stores your projects, /var/log contains error logs from failed analysis runs, /tmp is for intermediate pipeline files.

Absolute vs Relative Paths

Absolute Path

Starts from root (/)

/home/biouser/projects/genome_analysis/data.fasta

✓ Works from anywhere

Relative Path

From current location

./data.fasta
../results/output.txt
../../shared/reference.fa

. = current, .. = parent

Example: If you are in /home/biouser/projects/
  • cd genome_analysis (relative)
  • cd /home/biouser/projects/genome_analysis (absolute)
📝 Speaker Notes:
Absolute paths are complete addresses from root - like full postal addresses. Relative paths are directions from where you currently are. In bioinformatics scripts, use absolute paths for reliability; use relative for portability across different systems.

The Shell and Its Types

The shell is a command-line interface to interact with the OS

bash (Bourne Again Shell)

  • Most popular, default on many systems
  • Great for scripting pipelines
  • Command: #!/bin/bash

sh (Bourne Shell)

  • Original Unix shell
  • More portable, fewer features
  • Used in minimal environments

zsh (Z Shell)

  • Modern, feature-rich
  • Better auto-completion
  • Default on macOS

csh/tcsh (C Shell)

  • C-like syntax
  • Less common for scripting
  • Some legacy bioinformatics tools
$ echo $SHELL    # Check your current shell
📝 Speaker Notes:
The shell interprets your commands. Bash is standard for bioinformatics - most pipelines are bash scripts. The shell processes your commands, expands wildcards (*.fasta), handles variables ($HOME), and manages input/output redirection.

Creating & Deleting Files/Directories

Creating

# Create empty file
touch experiment_results.txt

# Create multiple files
touch sample1.fasta sample2.fasta sample3.fasta

# Create directory
mkdir genome_data

# Create nested directories
mkdir -p projects/2024/january/week1

# Create file with content
echo "Sequence data" > seq.txt

Deleting

# Remove file
rm old_data.txt

# Remove directory (must be empty)
rmdir empty_folder

# Remove directory with contents
rm -r genome_data/

# Remove with confirmation
rm -i important_file.txt

# Force remove (DANGEROUS!)
rm -rf folder/    # Use with caution!
⚠️ Warning: rm is permanent - no recycle bin!
📝 Speaker Notes:
touch creates empty files or updates timestamps. mkdir makes directories; -p flag creates parent directories too. rm removes files permanently - there's no undo! Always double-check before using rm -rf.

File Permissions Structure

-rwxrw-r-- 1 biouser staff 4096 Jan 15 10:30 script.sh
-         rwx      rw-      r--
│          │        │        │
│          │        │        └─ Others (all other users)
│          │        └────────── Group (users in file group)
│          └───────────────────── Owner (user who owns file)
└──────────────────────────────── File type (- = file, d = dir, l = link)

r = read (4)     w = write (2)     x = execute (1)
Example:
  • rwx = 4+2+1 = 7 (read, write, execute)
  • rw- = 4+2+0 = 6 (read, write)
  • r-- = 4+0+0 = 4 (read only)
  • rwxrw-r-- = 764
📝 Speaker Notes:
Every file has 3 permission sets: owner, group, others. Each set has 3 permissions: read (view contents), write (modify), execute (run as program). For files: r=read content, w=modify, x=run. For directories: r=list contents, w=add/delete files, x=enter directory.

Changing Permissions - Symbolic

Symbolic Notation (chmod)

# Who: u=user(owner), g=group, o=others, a=all
# Operation: + (add), - (remove), = (set exactly)
# Permission: r, w, x

# Give owner execute permission
chmod u+x script.sh

# Remove write from group and others
chmod go-w data.txt

# Set exact: owner=rwx, group=rx, others=r
chmod u=rwx,g=rx,o=r analysis.py

# Add read for everyone
chmod a+r readme.txt
Biotech Example:
  • chmod u+x run_pipeline.sh - Make pipeline executable
  • chmod go-rwx patient_data.txt - Restrict sensitive data
📝 Speaker Notes:
Symbolic method is human-readable. u/g/o specify who, +/-/= specify operation, r/w/x specify permission. Use when making specific changes. For bioinformatics: make scripts executable (u+x), protect sensitive data (go-rwx), share results (a+r).
Slide 8 of 16

Changing Permissions - Numeric

Numeric Notation

# Each digit: owner, group, others
# Calculate: r=4, w=2, x=1, add them

chmod 755 script.sh       # rwxr-xr-x
chmod 644 data.txt        # rw-r--r--
chmod 600 private.key     # rw-------
chmod 700 my_folder/      # rwx------
chmod 775 shared_dir/     # rwxrwxr-x

Common Patterns

  • 777 - rwxrwxrwx - Everyone everything (dangerous!)
  • 755 - rwxr-xr-x - Standard executable scripts
  • 700 - rwx------ - Private executable
  • 644 - rw-r--r-- - Standard data files
  • 600 - rw------- - Private data/keys
📝 Speaker Notes:
Numeric method is faster. Three digits: owner-group-others. Calculate: r(4) + w(2) + x(1). 755 means owner full control, group and others read/execute. 644 is standard for files. 600 for private files. 755 for scripts.

Processes in Linux

Process Commands

# View running processes
ps               # Your processes
ps aux           # All processes, detailed

# Real-time monitoring
top              # Press q to quit
htop             # Better (if installed)

# Find specific process
ps aux | grep python

# Kill process by PID
kill 1234
kill -9 1234     # Force kill

# Kill by name
killall python
pkill blast
Biotech Use: Monitor BLAST runs, kill stuck jobs, check resource usage
📝 Speaker Notes:
A process is a running program with unique PID. ps shows snapshot. top shows real-time. kill sends signals: default TERM asks nicely, -9 forces. Monitor memory-intensive aligners, kill runaway jobs, track parallel analyses.

Filters - grep

grep - Search Text

# Basic search
grep "pattern" file.txt

# Case-insensitive
grep -i "sequence" data.txt

# Show line numbers
grep -n "error" log.txt

# Count matches
grep -c ">" sequences.fasta

# Invert match
grep -v "comment" data.txt

# Recursive in directories
grep -r "TODO" project/
Examples:
  • grep ">" sequences.fasta - Extract FASTA headers
  • grep -c "ATGC" dna.txt - Count sequences
📝 Speaker Notes:
grep searches for patterns. -i ignores case, -n shows line numbers, -c counts, -v inverts. Use to extract FASTA headers, find genes, filter files, search logs. Essential for bioinformatics text processing.

Pipes - Command Chaining

Pipe (|) Examples

# Basic pipe
ls -l | grep ".txt"

# Multiple pipes
cat genes.txt | sort | uniq | wc -l

# Count FASTA sequences
grep ">" sequences.fasta | wc -l

# Find largest files
du -sh * | sort -rh | head -10

# Count per chromosome
grep -v "^@" aligned.sam | cut -f 3 | sort | uniq -c
Power: Combine simple tools into powerful workflows without intermediate files
📝 Speaker Notes:
Pipes connect commands - output of one becomes input of next. Build complex workflows incrementally. Essential for genomics: extract fields, filter, count. No intermediate files needed. Test each step, then combine.

Input/Output Redirection

Redirection Operators

# Output to file (overwrite)
command > output.txt

# Append to file
command >> output.txt

# Redirect errors
command 2> errors.log

# Redirect both stdout and stderr
command > output.txt 2>&1
command &> all_output.txt

# Input from file
sort < unsorted.txt

# Discard output
command > /dev/null 2>&1
Example: bwa mem ref.fa reads.fq > aligned.sam 2> bwa.log
📝 Speaker Notes:
Redirection controls data flow. > overwrites, >> appends. 2> redirects errors. &> redirects both streams. < reads input from file. Critical for bioinformatics: save results, log errors separately, chain analyses.

File Compression - tar & gzip

tar - Archive

# Create archive
tar -cvf archive.tar folder/

# Extract
tar -xvf archive.tar

# Compressed (.tar.gz)
tar -czvf archive.tar.gz folder/
tar -xzvf archive.tar.gz

gzip - Compress

# Compress
gzip large_file.txt

# Decompress
gunzip file.txt.gz

# View compressed
zcat file.txt.gz
zgrep "pattern" file.txt.gz
📝 Speaker Notes:
tar bundles files. Add z for gzip compression. FASTQ files compress 75-90% with gzip. Most bioinformatics tools read .gz directly. Use tar -czvf for backups, gzip for daily compression.

Regular Expressions with grep

Basic Regex

# Start/end of line
grep "^chr" genome.txt       # Starts with chr
grep "gene$" file.txt        # Ends with gene

# Character classes
grep "[0-9]" data.txt        # Any digit
grep "[ATGC]" dna.txt        # DNA bases

# Quantifiers
grep "A\{3\}" seq.txt        # Exactly 3 A's
grep -E "chr[0-9]+" file.txt # One or more digits
Biotech:
  • grep "^>" file.fa - FASTA headers
  • grep -E "chr[0-9XY]+" variants.txt - Chromosomes
📝 Speaker Notes:
Regex enables pattern matching. ^ is line start, $ is end. [ATGC] matches DNA bases. Use -E for extended regex. Essential for validating sequences, extracting IDs, parsing formats.

Best Practices

Do ✓

  • Use absolute paths in scripts
  • Comment your code
  • Test on small datasets first
  • Keep backups
  • Log errors: 2> errors.log
  • Use version control (git)

Don't ✗

  • Never use rm -rf without checking
  • Don't use chmod 777
  • Avoid spaces in filenames
  • Don't work on original data
  • Don't ignore errors
📝 Speaker Notes:
Professional practices prevent disasters. Test destructive commands. Document everything. Keep raw data read-only. Log everything. Use meaningful names. Backup before changes.
